What's inside

Everything ops in one place

Purpose-built for CloudOps engineers who need fast access to operational knowledge without switching tools.

📖

Runbooks

16 operational runbooks covering CPU, memory, disk, networking, and cloud-specific failure patterns — with PowerShell and Bash remediation scripts.

🔌

Integrations

32 cloud service connectors. View and manage all platform integrations — AWS, Azure, monitoring tools, and alerting pipelines — from a single panel.

🤖

Agents

Monitor 24 cloud agents across your infrastructure. Real-time status, health indicators, and configuration controls in one view.

🔍

Grouped navigation

Runbooks are organised into collapsible groups — CPU, Memory, Disk — so engineers find what they need without scrolling through flat lists.

🌐

No CDN dependency

All JavaScript assets (React, ReactDOM, Lucide) are served locally from inside the container. Works on air-gapped and restricted enterprise networks.

🚀

Zero configuration

One Docker command to run. No database, no auth service, no environment variables required. Just pull and go.

Architecture

Lightweight by design

Deployment

Run it anywhere

Any machine with Docker installed can pull and run the portal in seconds. No additional setup required.

Terminal

Pull and start the container:

$ docker run -d -p 5000:5000 --name hcms-portal gauravheychawladocker/hcms-cloud-portal:latest

Or use docker-compose:

$ docker-compose up -d

Then open http://localhost:5000 in your browser.

Replace localhost with the server IP when running on a remote host.
